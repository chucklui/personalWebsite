Company Project

The Student Information System is a web system to manage classes and their associated lectures, assessments, etc. The system is built with Django as backend framework and uses PostgreSQL to manage it's database data. During my intern, me and my partner helped implement an editable rainbow chart feature in SIS for the ease of staff to make changes to cohort items on the same page. In addition, we updated Django files and include a date library to allow API endpoints to accept appropriate data when updating rainbow chart via AJAX.

Warbler

Warbler is a mock-twitter clone application built with backend framework Flask and Jinjia. PostgreSQL is used to store the data and SQLAlchemy to manage database relationship. This app is similar to Twitter where users can signup/login and make "warbles". It also include authentication/authorization where bcrypt and session are being used to verify and store user information. Tests are also written for the views and unittest for the backend.

Jobly

Jobly is a mock-job board searching application. It is build with React in the front-end and Express in the back-end. PostgreSQL and SQLAlchemy are also being used to store data and maange the db relationship. Users can signup/login via bcrypt password and JsonWebToken to verify the user. Testing were implemented by Jest for unit testing and integration testing.
